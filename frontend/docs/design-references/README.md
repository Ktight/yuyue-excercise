## 视觉实施规则

- 先从参考稿提取颜色、字体、圆角、阴影、间距和响应式断点到现有 Token/Theme 层。
- 页面只使用语义变量，不直接散落参考稿颜色值。
- 桌面与移动端共用业务组件、领域模型、API 和适配器；只在布局和呈现组件上响应式变化。
- 不根据图片猜测尚未存在的业务接口或字段。
- 视觉改造不得改变 Phase 1–7 已验证的权限、路由、请求、Mock 和错误处理逻辑。
- 每批视觉修改后运行 `check:all` 和对应页面视觉回归。
