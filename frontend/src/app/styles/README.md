# 样式分层

全局样式只能通过 `index.css` 进入应用，固定顺序为：

1. `reset`：浏览器差异归一化；
2. `tokens`：原始颜色、字号、间距、圆角、阴影等稳定变量；
3. `theme`：页面、内容、操作等语义变量，可替换主题；
4. `base`：`body`、链接、焦点和选区等元素基础表现；
5. `layout`：无业务含义的容器、栈、排列和响应式规则；
6. `components`：原生控件的统一基础行为；
7. `utilities`：少量显式工具类。

维护规则：

- 业务组件优先使用语义变量，不新增硬编码品牌颜色；
- Feature 样式继续使用 `<style scoped>`，不得写全局覆盖；
- 不使用 `!important`，唯一例外是具有明确强制语义的工具类；
- 主题只覆盖语义变量，不复制业务组件 CSS；
- 现有变量名称保留为兼容契约，迁移时不要求一次性修改所有页面；
- API、Store、Schema、路由和事件逻辑不得依赖样式层。
