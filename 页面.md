## 3. 页面详细说明

### PAGE-001 登录

- 内容：产品名“瑜悦练”、副标题“瑜伽馆管理系统”、登录卡片。
- 字段：手机号、密码。
- 功能：前端校验、字段错误、服务端错误、提交中禁用、登录成功后按角色进入对应首页。
- 设计重点：这是品牌第一印象，需要同时体现瑜伽气质、系统可信度和清晰的表单操作。

### PAGE-002 修改密码

- 内容：修改密码标题、密码表单、完成提示。
- 字段：当前密码、新密码、确认新密码。
- 功能：字段校验、错误提示、提交状态、成功后切换为完成信息。
- 设计重点：强化安全感，明确区分输入、错误、提交和成功状态。

### PAGE-003 公司管理

- 内容：页面标题、新建公司按钮、公司列表。
- 功能：加载、错误重试、空状态、查看公司状态、进入详情；只有超级管理员显示新建入口。
- 设计重点：管理型列表要保持高信息效率，不应设计成营销页面。

### PAGE-004 新建公司

- 字段：公司名称、地址、联系人、联系电话。
- 功能：必填校验、提交、保存中、错误反馈，成功后返回公司列表。
- 设计重点：标准业务表单，字段层级、必填提示和反馈必须统一。

### PAGE-005 公司详情

- 内容：公司名称、地址、联系人、联系电话、公司状态、所属门店列表。
- 功能：按权限只读或编辑公司；启用/停用公司；点击所属门店进入门店详情。
- 状态：加载、错误、正常展示；不同权限产生编辑和只读两种视图。
- 设计重点：同一页面同时容纳公司资料、危险/状态操作和关联门店。

### PAGE-006 门店管理

- 内容：页面标题、新建门店按钮、门店列表及状态。
- 功能：加载、错误重试、空状态、进入门店详情、新建门店。
- 设计重点：与公司列表保持同一列表体系，同时能够快速识别门店地址和状态。

### PAGE-007 新建门店

- 内容：所属公司选择器、门店资料表单。
- 字段：所属公司、门店名称、地址。
- 功能：先加载公司选项；加载失败提示；提交和错误反馈；成功后返回门店列表。
- 设计重点：清楚表达门店和公司的从属关系。

### PAGE-008 门店详情

- 内容：门店名称与地址、门店编辑表单、启停按钮、场地列表、场地新建/编辑表单。
- 场地字段：场地名称、容量、说明。
- 功能：加载和重试；按权限编辑或启停门店；查看、启停、编辑、新增场地。
- 设计重点：这是信息最复杂的代表页面之一，应分清门店信息、状态操作、场地列表和场地编辑四个区域。

### PAGE-009 课程模板

- 内容：页面标题、新建模板按钮、课程模板列表。
- 功能：加载、错误重试、空状态、点击进入详情、新建模板。
- 设计重点：课程卡片应突出名称、类别、难度、时长、容量和状态，不得让装饰图片压过业务信息。

### PAGE-010 新建课程模板

- 字段：课程名称、类别、难度、时长、容量、描述。
- 规则：时长至少 15 分钟；私教类别容量固定为 1。
- 功能：校验、保存中、错误反馈，成功后返回模板列表。
- 设计重点：数值、枚举和描述字段需要形成清晰的表单分组。

### PAGE-011 课程模板详情

- 内容：模板名称、状态标签、启停按钮、完整课程模板表单。
- 功能：加载、错误、编辑保存、启用或停用。
- 设计重点：状态标签与状态操作要清楚，但不能与主保存动作争夺层级。

### PAGE-012 / PAGE-015 学员管理

- URL：管理员 `/admin/students`；训练师 `/trainer/students`。
- 内容：页面标题、新建学员按钮、学员卡片列表。
- 功能：加载、错误重试、空状态、进入学员详情、新建学员。
- 复用说明：两个 URL 当前使用同一个 Vue 页面组件，不是两个独立视觉模板。
- 当前限制：页面内部跳转目前写向 `/admin/students/...`；这是后续需要验证的角色路由适配点，视觉设计不得借此修改业务路由。

### PAGE-013 / PAGE-016 新建学员

- URL：管理员 `/admin/students/new`；训练师 `/trainer/students/new`。
- 内容：基本信息、会员卡、训练目标三个表单区。
- 字段：姓名、手机号、性别、生日、会员类型、到期日期、目标描述、备注。
- 功能：必填校验、保存中、失败反馈，成功后返回列表。
- 复用说明：两个 URL 使用同一个页面组件。
- 设计重点：这是复杂表单代表页，需要兼顾桌面双列和移动端单列。

### PAGE-014 / PAGE-017 学员详情

- URL：管理员 `/admin/students/:id`；训练师 `/trainer/students/:id`。
- 标签：档案信息、身体评估、课时记录、训练规划。
- 已实现：加载学员、错误提示、查看和编辑档案信息。
- 尚未实现：身体评估、课时记录、训练规划只显示“后续 Phase 接入”。
- 复用说明：两个 URL 使用同一个页面组件。
- 设计重点：可以设计完整的标签导航和占位状态，但不能虚构三个尚未实现模块的数据与操作。

### PAGE-018 用户管理

- 内容：创建用户入口、搜索和角色筛选、用户列表、分页。
- 功能：按姓名或手机号搜索、按角色筛选、URL 保留筛选和页码、上一页/下一页、加载、错误重试和空状态。
- 每页数量：20 条。
- 设计重点：这是验证信息密度、筛选器、列表、分页和状态反馈的核心页面。

### PAGE-019 创建用户

- 字段：姓名、手机号、密码、角色、所属门店（部分角色显示）。
- 规则：门店管理员必须选择门店；训练师可以关联门店；不适用角色不显示门店字段。
- 功能：加载门店选项、字段级校验、服务端字段错误映射、提交中禁用、创建成功返回列表。
- 设计重点：动态字段出现时页面布局不能跳动混乱，角色与门店关系要容易理解。

### PAGE-020 管理员工作台

- 当前内容：标题“管理员仪表盘”、欢迎语、等待功能模块加载提示。
- 当前状态：占位页面，没有可确认的经营指标、快捷操作或图表。
- 设计边界：可探索工作台的视觉框架，但所有新增数据卡片必须标为“概念提案”，不能计入已实现功能。

### PAGE-021 训练师工作台

- 当前内容：标题“训练师工作台”、欢迎语、等待功能模块加载提示。
- 当前状态：占位页面，没有已实现的排课、学员提醒或今日课程功能。
- 设计边界：不得把合理猜想描述成现有功能。

### PAGE-022 学员首页

- 当前内容：标题“我的瑜悦练”、欢迎语、等待功能模块加载提示。
- 当前状态：占位页面，没有已实现的课程、练习计划或个人数据内容。
- 设计边界：可作为品牌方向探索，但概念内容必须和真实功能清单分开。

### PAGE-023 无权限

- 内容：403、没有访问权限、当前账号不能访问、返回我的首页。
- 功能：清楚解释权限问题，并提供恢复路径。
- 设计重点：语气友好、错误原因明确、主操作突出。

### PAGE-024 页面不存在

- 内容：页面未找到、检查地址提示。
- 功能：当前只显示提示，没有显式返回按钮。
- 设计重点：可以优化视觉表达，但增加导航操作前应确认是否属于本轮功能修改。

## 4. 三个布局壳

布局壳不单独计算为业务页面，但视觉方案必须覆盖：

1. 管理员布局：管理端页面的导航和内容容器。
2. 训练师布局：训练师工作台及训练师学员页面容器。
3. 学员布局：学员首页及未来学员自助功能容器。

主题应通过公共布局和语义变量生效，不应为每个角色复制一套无关的 CSS。

## 5. 可复用的页面模板

21 个独立页面不等于需要设计 21 套完全不同的结构。建议归纳为：

| 模板 | 对应页面 |
|---|---|
| 品牌认证表单 | 登录、修改密码 |
| 管理列表 | 公司、门店、课程模板、学员、用户列表 |
| 新建/编辑表单 | 公司、门店、课程模板、学员、用户创建与详情编辑 |
| 主体与关联资源详情 | 公司详情、门店详情 |
| 标签式档案详情 | 学员详情 |
| 角色工作台/首页 | 管理员工作台、训练师工作台、学员首页 |
| 系统状态 | 403、404 |

## 6. 第一轮商业视觉评审建议

老板确认风格前，建议三套主题各制作以下代表页面，而不是重复制作全部页面：

1. 登录：品牌第一印象。
2. 管理员工作台：整体框架和导航；必须标明当前为概念/占位。
3. 用户管理：高密度管理列表。
4. 新建学员：复杂业务表单。
5. 学员详情：档案和标签结构。
6. 门店详情：多区域详情与内嵌场地管理。
7. 课程模板列表或详情：内容卡片和状态表达。
8. 403/404：系统状态体验。

确认最终主题后，再把同一套语义变量和组件规范扩展到剩余页面。

## 7. 事实边界与待验证项

- 本文统计的是前端页面事实，不承诺真实 API 已完成。
- 管理员和训练师的学员路由复用页面组件，但页面内跳转是否完全保持角色前缀仍需功能测试。
- 三个工作台是明确占位页，不能作为已完成业务功能展示。
- 学员详情中的身体评估、课时记录、训练规划尚未实现。
- 视觉设计不得修改接口字段、角色权限、表单规则和路由逻辑。
- 若路由或页面后续新增、删除或变更，本文件也需要同步更新日期和统计数字。
